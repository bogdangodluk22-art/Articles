// 01 — OVERVIEW

What the system does

Four modules that together replace an entire sales department — from finding the first contact to onboarding a signed client.

🔍
Automatic lead sourcing and enrichment
Every week the system parses Google Maps and Telegram groups on its own, filters by quality (rating, number of reviews, presence of a contact), deduplicates, and enriches emails via the venues' websites. A new week — new leads with no manual action.
💬
Personalized cold outreach
GPT-4o generates a unique first message for each lead — taking into account the name, city, category and rating. Telethon sends it through a personal Telegram account: not a mass mailing, but a personal message with human-level conversion.
🤖
Live AI dialogue with memory and RAG
The agent Oleksiy (GPT-4o) remembers the entire conversation via Supabase and queries the vector knowledge base in seconds. It adapts the tone to the client's status in the funnel. An objection handled with concrete ROI — not a template.
🎙️
Voice agent as a Web App in Telegram
Three ElevenLabs agents open right inside Telegram — no browser, no redirects. The client taps a single button and talks in a lifelike voice. After the call — an automatic GPT analysis: interest, objections, next action.

// 02 — ARCHITECTURE

How it works — the full flow

Six stages from the first parse to a signed client. Each one is a separate workflow, each one is fully automatic.

01
🔍 Lead parsing — the system finds clients on its own
Every Monday at 9:00 two parsers run in parallel. Block 1 launches the Apify Google Maps Scraper with filters: rating 4.0+, at least 50 reviews, a phone number present. Found leads are deduplicated against Airtable and saved to PARSED. Block 2 is triggered automatically — it opens each venue's website, finds the email and adds it to the card (noise filter: noreply, support@, admin@ — ignored).
💡 Block 1b in parallel: Python + Telethon parses HoReCa Telegram groups. n8n runs the script via SSH, the results come in via a webhook. Two lead sources — two funnel streams.
Apify Google Maps Python + Telethon Airtable PARSED Email Enrichment n8n Schedule
02
📤 First contact — a personalized message through a personal account
Block 3 (Email Outreach) and Block 4 (TG Cold Outreach) run 3 times a day. Before each send, the system checks the daily limit via the Supabase daily_outreach_log (10 messages/day, batches of 4). GPT-4o generates a unique text for each venue — name, district, category, rating. The Telethon agent sends it through Oleksiy's personal TG account (not a bot account). Between sends — a 45-second pause.
💡 A personal Telegram account vs a bot: higher trust, higher open rate, no "bot" label. This is a fundamental difference in first-contact conversion.
GPT-4o personalization Telethon VPS Gmail Outreach Supabase Limit Log
03
🤖 Live dialogue — the AI agent Oleksiy runs the conversation
The client's reply comes in via a webhook to Block 4 Reply. A Switch node determines the branch by outreach_status: A (first reply, set Replied + create IN_PROGRESS), B/C/D (continuation of the dialogue). Each branch loads the last 10 messages from the Supabase dialogue_history and queries the vector knowledge base via pgvector RAG. GPT-4o responds in context — it knows what the client said, knows the product, adapts the tone.
💡 RAG (Retrieval-Augmented Generation): instead of a hardcoded prompt — a semantic search over the knowledge base. Update a document in Google Drive → the system re-builds the embeddings automatically. The knowledge is always up to date.
GPT-4o Agent RAG pgvector dialogue_history Switch A/B/C/D Supabase
04
📲 Telegram Bot + Web App — a voice agent right in the messenger
The client enters @resstrolux_bot. Block 5A (75 nodes) handles all events: /start, demo_click, offer_click, menu, text. On /start — a Supabase GET determines whether the client is new (Welcome + WebApp button) or returning (menu). The "Talk to the agent" button opens the ElevenLabs Web App right inside Telegram — a full-fledged voice AI with no browser. The system determines which agent to show by bot_stage: Demo (#2) → Consultation (#3) → Support (#4).
💡 A Web App in Telegram is not just a link. It is a native mini-app inside the messenger. The client is already in a secure, trusted environment. The conversion into a conversation is an order of magnitude higher than "open a website".
Telegram Bot API Web App (Mini App) ElevenLabs Conv. AI Guard nodes bot_stage routing
05
⏱️ Follow-up Scheduler — automatic reminders on a timer
Block 5C runs every 15 minutes and checks 4 conditions in Supabase in parallel. FU#1: the client tapped "Demo" more than 30 min ago and followup_1_sent=false → send a message with loss figures and a [Show what we offer] button. FU#2 (30 min after offer_clicked) → cases and ROI with the Consultation agent button. FU#3 (after consultation_booked) → a personal follow-up. FU#4 → a welcome for a new subscriber. After each one — PATCH followup_N_sent=true. It will never send twice.
💡 A Schedule Trigger instead of Wait nodes — more reliable for a VPS. A Wait node stores its state in n8n's SQLite, which can be cleared on restart. Schedule always reads the current state from Supabase.
Schedule 15 min Supabase conditions 4 parallel FUs Airtable sync
06
💳 Payment, onboarding and a personal account
The client chooses a plan on the product website (Lovable). The Site Webhook (/restroai-site) handles the payment_success event: parses plan, tg_chat_id, amounts → UPSERT into Airtable CLIENTS → PATCH the IN_PROGRESS status to subscribed → PATCH the Supabase contacts. The client gets access to their personal account (auth: phone + SHA-256 password). Block 5G Get Agent Config now returns agent #4 Support with the full context: name, restaurant, plan, subscription date — all passed via dynamic variables before the first word.
💡 The voice support agent knows the client before the conversation even begins. It's not "how can I help you?" — it's "Hi, Andriy! I see you signed up for Setup+Sub on April 15. How can I help?"
Site Webhook Airtable CLIENTS Client Portal Agent #4 Support Dynamic Variables

// 05 — LAUNCH

What's needed to set it up

Five steps from zero to a fully working B2B sales funnel.

1
Prepare the infrastructure A VPS (Hostinger or similar, from $6/mo), Docker Compose for n8n, a Supabase project with schema v3 (the contacts, dialogue_history, daily_outreach_log, documents tables for pgvector). A Telegram bot via BotFather, a domain registration for the WebApp.
2
Set up Airtable and the lead sources Create 6 tables in Airtable (PARSED, PARSED_TG, IN_PROGRESS, CLIENTS, CALL_ANALYSIS, CONSULTATIONS) with the field schema. Connect Apify for Google Maps parsing. Set up the Python Telethon agent on the VPS with an account and a session.
3
Import and activate the workflows Upload the 14 workflow JSON files into n8n. Configure the credentials: OpenAI, ElevenLabs, Airtable, Supabase, Telegram Bot, Gmail. Check the webhook URLs — they are tied to the specific n8n domain.
4
Create the ElevenLabs agents and upload the knowledge base Three agents in the ElevenLabs Dashboard (Demo, Consultation, Support) with prompts, tools (send_demo, send_site, send_support), Analysis criteria and Widget settings. Upload the Knowledge Base document to the Google Drive folder — the system auto-embeds it into pgvector.
5
Test run and activation Test the full flow: /start the bot → Web App agent → a test call → check CALL_ANALYSIS in Airtable → test the follow-up scheduler → test the payment webhook. Activate all workflows. The first parse will run automatically the following Monday at 9:00.
