01 · Problem

Content without a system —
is a lottery, not a strategy

Every producer, expert and entrepreneur has at least once spent weeks on content — and gotten nothing.

Monday, 11:40 PM. You're sitting there trying to figure out what video to make. You open up your competitors, scroll through TikTok, check YouTube Trends. An hour later you have 40 tabs open, notes in five different places, and a complete feeling that you've gotten nowhere. In the end, you film whatever seemed okay. The video gets 300 views. Again.

This isn't a question of talent. It isn't even a question of ideas. It's a question of a system.

Producers who grow don't pull topics out of thin air. They analyze what works for their competitors, track trends before they catch fire, and build a content plan on data, not on gut feeling.

The problem is that this process used to take dozens of hours a month. Or it cost a team of analysts. Or it simply didn't happen at all — because there wasn't enough time.

The average producer spends 8–12 hours a month on manual analysis of channels, competitors and trends. And 70% of that time produces an inaccurate or outdated picture. I decided to automate this process completely.

0 AI Character

Osiris — a 3D robot with memory and RAG

The skyscraper metaphor

The core UX idea is a skyscraper with floors. The user rides an elevator between floors: the doors close, a movement animation plays, the doors open into a new room. Each room has its own atmosphere, color and character.

This isn't just aesthetics. It's funnel logic: floors 1 and 2 collect data → floors 3 and 4 generate ideas → floor 5 builds the strategy. All the data is stored in Supabase and passed between the tools.

05 · Osiris

The brand's third hero.
A smart machine.

"Osiris isn't a chatbot. It's an AI character that knows where you are in the platform, remembers your entire history, and responds like a senior analyst."

Babylon.js + three GLB models: base.glb (rotation), breathing.glb (active state), touch.glb (typing a response). Hosted on GitHub Pages, loads from any device.

GPT-4o via an n8n webhook. RAG on pgvector — Osiris reads the knowledge base through semantic search. Full dialogue memory in Supabase. It knows which tool is currently active via the tool field in user_state.

10 · Glossary

Glossary of terms

If something is unclear — expand it and you'll find an explanation.

📖 Expand the glossary of terms ▼
n8nA self-hosted automation platform. Like Zapier, but with no limits and no monthly charges per run. Installed on your own server.
WebhookA URL that waits for an incoming request. When the website sends data, the webhook receives it and triggers the workflow.
ApifyA web-scraping platform. You run an "actor" (a script) — it collects data from YouTube, Instagram, TikTok and returns JSON.
TavilyAn AI search API. Unlike Google, it returns clean data with no ads or SEO junk. Used for trend analysis.
SupabaseAn open-source alternative to Firebase. A PostgreSQL database with a REST API, auth, realtime. GoTrends uses it as the single source of truth.
pgvectorA PostgreSQL extension for storing vectors. It enables semantic search — finding similar content by meaning rather than by words.
RAGRetrieval-Augmented Generation. A technique where the AI searches for relevant information from a knowledge base before answering. Osiris uses RAG for accurate answers.
GPT-4oOpenAI's latest multimodal model. GoTrends uses it for analysis, idea generation, building strategies and Osiris's dialogues.
Babylon.jsA JavaScript engine for 3D graphics in the browser. It loads GLB models and renders Osiris with animations right on the page.
Telegram WebAppA Telegram API that lets you open full web pages inside the messenger. GoTrends opens inside Telegram without leaving the app.
splitInBatchesAn n8n node for processing a list one item at a time. Floor 02 uses it to analyze competitors sequentially.
tool fieldA field in Supabase (the user_state table) that records where the user currently is: fallback, osiris, channel_analysis, competitor_analysis, etc.
