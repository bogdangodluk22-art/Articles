// 02 — ARCHITECTURE

How it works

From the client's first word to a card in the CRM — 5 steps

01
The client starts the conversation
The client writes in the chat on the website or makes a call — and is connected to Daryna right away. She greets them, finds out which procedure they're interested in, talks about prices and answers questions.
💡 Daryna knows the entire price list, the FAQ and the sales script. She never gets tired and never makes mistakes with prices.
ElevenLabs Conversational AI voice + text Ukrainian language
02
The agent collects the data and sends a webhook
When the client is ready to book — Daryna collects the name, phone, service and convenient time. ElevenLabs sends this data to n8n via a webhook.
💡 For voice calls — after the conversation ends, ElevenLabs separately sends the full transcription and an audio file.
webhook POST submit_lead tool call post_call_transcription
03
n8n parses the event type and runs the right branch
The workflow determines what came in — an instant lead from the chat, a voice call, or a transcription of a text conversation — and runs the corresponding processing branch.
💡 Three branches: submit_lead (chat) → voice_call (voice) → chat_transcript (chat transcription). Each branch does its own job without overlap.
n8n Switch node event_type detection text_only flag
04
Gemini analyzes the conversation and writes a summary
Google Gemini receives the transcription and performs two tasks: it scores the assistant's quality of work against 8 criteria (QA) and writes a short summary of the conversation for the manager.
💡 For voice calls, the MP3 audio file is stored on Google Drive — the manager can listen to it if needed.
Gemini 2.5 Flash QA scoring (8 criteria) ai_summary generation Google Drive MP3
05
The data is saved to the CRM — the manager gets a notification
The lead card and the call record are saved to Airtable with a full link between them. The manager receives a single Telegram message with all the information — nothing extra.
Airtable upsert lead_id → call link Telegram Bot API one notification only

// 05 — LAUNCH

What's needed to set it up

Five steps from zero to a fully working system

1

Set up the ElevenLabs agent

Create the Daryna agent in ElevenLabs, upload the clinic's knowledge base (price list, FAQ, script), connect the webhook to n8n and configure the tool call for passing the form data.

2

Prepare Airtable

Create the Leads and Calls tables with all the fields according to the schema. Obtain the API key and table IDs to connect to n8n.

3

Connect Google Drive and Gemini

Authorize Google Drive OAuth2 and create a folder for storing the audio. Obtain a Gemini API key for analyzing the quality of conversations.

4

Create a Telegram bot and configure notifications

Register a bot via BotFather, obtain the token and Chat ID. Configure the message format to suit the manager's needs.

5

Import the workflow into n8n and activate it

Upload the workflow JSON file, fill in all the credentials, test each of the three branches on sample data and enable auto-run.
