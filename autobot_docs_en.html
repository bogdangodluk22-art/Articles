// 01 — OVERVIEW

What the system does

Four independent modules that together form a single chain: from the client's first message to a warm lead card in the manager's hands.

🤖
AI car matching in Telegram
The bot guides the client through 7 questions or accepts free-form text. GPT-4o analyzes the profile and selects 3–4 cars from real listings — with photo, description and link.
🌐
Voice assistant on the website
An ElevenLabs-powered widget answers customer questions in a lifelike voice, 24/7. Any question about services, prices or the process — an instant answer with no manager involved.
📨
Automatic follow-up
Every day the system checks unbooked leads and sends a personalized message — with a photo of the exact car the client was interested in, and a call to action.
📸
Auto-posting to Instagram
Every day at 10:00 the system generates a topic, picks a photo via Unsplash and writes a ready post. The manager gets a preview in Telegram and decides whether to publish or not.
// the system's core value
⚡️ The manager receives a warm lead with a ready profile — instead of a cold call into the void
The client completed the questionnaire themselves, chose the car themselves, and tapped "Contact me" themselves. The manager sees: name, budget, car of interest, convenient time. The deal can be closed right away.
// 02 — ARCHITECTURE

How it works — the full flow

The path from first contact to a closed deal. Every step is automatic, every action is logged in the CRM.

01
🚪 Entry point — website or Telegram
The client lands on the website and talks to the voice assistant (ElevenLabs), or goes straight to the Telegram bot via a link from Instagram / the website. First contact — without a manager.
💡 The voice assistant knows the answers to all typical questions about the services: what car matching is, how much it costs, how the inspection works — and answers in natural language.
ElevenLabs Voice Widget Telegram Bot API Instagram / Site Traffic
02
🧠 Session initialization and mode selection
On the first message the bot reads or creates a session in Supabase. The main Switch router determines the state: new user → greeting, returning user → continuation. The client chooses a mode: "Help me pick a car" (questionnaire) or a free-text request.
💡 All memory is stored in Supabase — if the client returns a week later, the bot remembers their profile and can offer to continue with the same parameters.
Supabase: GET session n8n Switch Node (8 outputs) POST: session initialization
03
📋 Questionnaire or free request → AI analysis
Mode 1 — the bot asks 7 questions: city, driving frequency, purpose, brands, budget, priorities, new or used. Each answer is saved to Supabase via a PATCH request. Mode 2 — the client writes in free text and GPT recognizes the parameters on its own. After collecting the data, GPT-4o produces 3–4 personalized recommendations in JSON format.
💡 The AI doesn't just filter by price — it takes the client's lifestyle into account. "Family with kids + out-of-town trips + reliability" → Toyota RAV4, not a BMW.
OpenAI GPT-4o Supabase: PATCH answers 7-step Quiz Flow JSON parsing
04
🔍 Listing search + car card
When the client picks a car from the recommendations, the system runs Tavily to search for real listings on auto.ria and other marketplaces. In parallel, Unsplash picks a car photo. The client receives a card: photo + name + year + price + link to the listing.
💡 If none of the options fit — the "More options" button triggers a new GPT request with updated recommendations, without resetting the whole flow.
Tavily Web Search Unsplash API Telegram: sendPhoto Supabase: PATCH chosen_car
05
📞 CTA — Consultation booking + CRM
After receiving the recommendations, the bot offers to book a consultation. The client leaves their name and phone number — the data is saved to Airtable (Quiz Leads or Free Leads) and Supabase. The manager receives a Telegram notification with the full lead card: name, phone, AI recommendations, chosen car, convenient time.
Airtable: Quiz/Free Leads Telegram: notification Supabase: user profile
06
🔄 Follow-up — Automatic reactivation
Every day at 18:00 the system reads Airtable and finds leads who haven't booked and haven't received a follow-up in less than 3 days. GPT generates a personalized message for each — mentioning the specific car they were interested in. After sending, the last_followup_at field is updated.
💡 This is not a mass mailing — every message is unique. "Hi, Oleksiy! You were interested in the Toyota RAV4. Prices have changed a bit — want to see the current options? 👀"
Schedule Trigger 18:00 OpenAI: personalization Airtable: PATCH status

// 05 — LAUNCH

What's needed to set it up

Five steps from a blank state to a fully working system.

1
Telegram bot and credentialsCreate a bot via @BotFather → get the token → add it to n8n as a Telegram credential. Configure the Webhook URL pointing to the n8n instance. This is the entry point for all messages from clients.
2
Supabase — two tablesCreate a Supabase project → the sessions table (chat_id, current_step, answers JSON, chosen_car, timestamps) + the site_bookings table. Add the Supabase URL and API key to n8n. Supabase is the bot's operational memory between sessions.
3
Airtable — the CRM baseCreate a "Car Matching" base → four sheets: Quiz Leads, Free Leads, Site Leads, Instagram Posts. Add an Airtable Personal Access Token to n8n. This is where the manager sees all leads and manages statuses.
4
External service API keysOpenAI API key (GPT-4o for matching and content generation) + Tavily API key (listing search) + Unsplash Client ID (photos) + ElevenLabs API key (voice assistant on the website). All keys are added to n8n credentials.
5
Import workflows and testingImport all 4 JSON files into n8n → activate → do a test run of each workflow. Verify that the bot responds, leads are saved to Airtable and Supabase, and the manager receives notifications.
