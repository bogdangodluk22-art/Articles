// 01 — OVERVIEW

What the system does

Four key modules that turn any guest request into a confirmed reservation — around the clock and without the human factor.

🎙️
Voice + text AI agent
The guest talks to the agent by voice or text right on the restaurant's website. Claude understands free-form requests — "a table for two on Saturday evening" — and holds the conversation in natural language.
📅
Real-time availability check
The system instantly checks Airtable: which tables are free for the requested date and time, which zone — terrace, main hall or VIP. If a table is taken, it offers an alternative.
🍽️
Menu pre-order
The guest can pre-order dishes right in the chat — the agent knows the full menu, can recommend dishes and answer questions about ingredients and allergens.
📊
Real-time manager dashboard
A separate web interface for the administrator: a floor plan with the live status of every table, a list of all reservations for the day, status management — everything pulled from Airtable.
// the system's core value
⚡ From request to confirmed reservation — less than 2 minutes. 24/7
The guest wrote or spoke — the system checked, recorded and confirmed. The administrator never picked up the phone. Not a single missed reservation — even at 3 AM on a weekend.
// 02 — ARCHITECTURE

How it works — the full flow

The path from the guest's first word to a record in the database and a notification to the manager. Every step is automatic.

01
🌐 Entry point — the restaurant's website
The guest visits the Villa Marea website (Base44). In the corner — a chat widget with two modes: text chat or a voice call via ElevenLabs. First contact happens without an administrator — the agent is always online.
💡 Voice mode uses ElevenLabs Conversational AI — a natural, lifelike voice, not synthetic TTS. The guest can simply speak, as if on the phone.
ElevenLabs Voice Widget Base44 Website Chat Interface
02
🧠 The AI agent receives the request and detects intent
The guest's request (text or voice transcript) arrives via a Webhook to n8n. The Claude API analyzes the message: it determines the request type — a new reservation, a cancellation, a pre-order, or a status check. The agent maintains the conversation context and clarifies details if something is missing.
💡 A Switch router in n8n distributes requests into 4 independent workflow branches: new reservation (Site / Agent), cancellation, menu pre-order, availability check.
Claude API (Anthropic) n8n Webhook Switch Router (4 outputs)
03
🔍 Table availability check
n8n makes a search query to the RESERVATIONS table in Airtable — looking up all reservations for the given date and time slot (1.5 hours). A code node analyzes the response: which of the 10 tables are free, which zone fits the number of guests. T01–T05 are for 1–2 guests, T06–T10 are for 3–5 guests.
💡 Booking slots are fixed: 12:00, 13:30, 15:00, 16:30, 18:00, 19:30, 21:00. The system automatically determines the correct slot based on the guest's preferred time.
Airtable: Search RESERVATIONS n8n Code Node Switch: Is Available
04
✅ Creating or updating a reservation
If the table is free — n8n checks whether a record already exists for this guest (by phone number). If not — it creates a new row in Airtable with all the details. If the guest is already in the database — it updates the existing record (upsert logic). The status is set to "Pending confirmation".
Airtable: Create Booking Airtable: Update Booking Airtable: Find by Phone
05
🍽️ Menu pre-order (optional)
After the table is confirmed, the agent offers to place a pre-order. The guest can choose dishes from the full Villa Marea menu — Claude knows all the items, prices and ingredients. The order is linked to the reservation through the Pre-Order field in Airtable.
💡 Pre-order is a separate workflow branch (Telegram Pre-Order). It is stored in the reservation field and is available to the administrator in the dashboard before the guest arrives.
n8n: Pre-Order Branch Airtable: Update Pre-Order Claude: Menu Knowledge
06
📲 Confirmation to the guest + notification to the manager
The guest receives a confirmation with the reservation details right in the chat — table, zone, time, date. At the same time, the manager receives a Telegram notification with the full summary: guest name, phone, table, time, number of guests and the pre-order if any. The manager's dashboard (Base44) updates in real time.
Telegram: New Booking 🟢 n8n: Voice Summary Base44: Manager Dashboard

// 05 — LAUNCH

What's needed to set it up

Five steps from zero to a fully working booking system.

1
Airtable — database setupCreate the "Villa Marea" base → the RESERVATIONS table with all the fields (guest_name, phone, table_id, zone, date, time_slot, guests_count, status, pre_order, notes). Add an Airtable Personal Access Token to the n8n credentials. The database is the single source of truth for the whole system.
2
n8n — importing and configuring the workflowImport the system's JSON file into n8n → connect the credentials: Airtable API, Telegram Bot Token, Claude API key. Activate the Webhook trigger — it accepts requests from ElevenLabs and the website. Verify that all 4 workflow branches (site, agent, cancel, pre-order) are active.
3
ElevenLabs — voice agent on the websiteCreate a Conversational AI agent in ElevenLabs → connect it to the n8n Webhook as the endpoint for the transcript. Embed the JS widget on the Base44 website. Configure the voice, language (Ukrainian) and the agent's system prompt with information about the restaurant and the menu.
4
Base44 — restaurant website and manager dashboardTwo separate projects in Base44: the public Villa Marea website (with an integrated chat widget and booking form) and the private manager dashboard (floor plan + reservation list). Both are connected to Airtable via the API and update in real time.
5
Testing and handoverA test run of each scenario: a new reservation through the website, a voice booking, a cancellation, a pre-order. Verify that records appear in Airtable, the manager receives Telegram notifications and the dashboard shows up-to-date data. After testing — hand over the access credentials to the client.
